
- **コンソール接続**\
コンピュータからシリアルケーブルなどを介して機器に接続し、文字による対話的な操作を行う。ルータなどのネットワーク機器の管理でよく用いられる。イーサネットポートの他に管理用のポートがあり、内部の制御システムを外部の別の機器から操作できる。コンピュータ側からはターミナルソフトなどで機器にアクセスし、管理者としてログインしてCLIで設定の表示や変更、再起動を実施できる

- **マネジメントポート**\
ルータやネットワークスイッチが備える通信ポート(端子)の1つで、コンピュータと直接繋いで管理者が操作を行うためのもの。管理用コンピュータとLANケーブルで直に繋いで通信する。機器内部の管理システムにアクセスでき、CLIで対話的に設定変更などの操作を行うことができる。GUIから管理メニューにアクセスできる機種もある

- **管理インターフェイス**\
デバイスにリモートでアクセスする主なインターフェイス。通常は帯域内ネットワークに接続されていないが、代わりにデバイスの内部ネットワークに接続される。管理者は管理インターフェイスを使用して、ネットワーク上のデバイスにSSHでアクセスできる。設定にはIPアドレス、プレフィックス、ネクストホップなどの情報が必要

- **境界(*perimeter*)FW**\
プライベートネットワークとパブリックネットワークの間の境界を保護し、確立するセキュリティアプリケーション。不要(不審)なデータがネットワークに侵入することを防ぎ、ネットワークに侵入しようとする各データパケットをスキャンして、サイバー攻撃や悪意のあるトラフィックから保護する

- **ttyコマンド**\
LinuxなどのUNIX系OSで標準的に用いられるコマンドの1つで、現在環境における標準入出力デバイスを表示するコマンド。文字の入出力を行う端末をファイルシステム内のデバイスファイル名で表示する。ttyで始まるものは物理的に接続された端末、ptsで始まるものはsshやTelnetなどを経由してネットワーク越しに接続された仮装端末を表す

- ***Fail Open***\
システム障害や故障時に、サービスや機能を停止せず、できるだけ利用者に影響を与えない形で動作を継続させる設計原則。可用性や継続性を重視するシステムにおいて、障害発生時にサービスを完全に停止はせず、部分的では利便性を保つことを目的とする

- ***Fail Close***\
システム障害や故障時に、システムを強制的に停止させることを目指す設計思想。安全性や信頼性を重視する場合に選択される

- **PPPoE**(*PPP over Ethernet*)\
PPPをイーサネットを通じて使えるようにした仕様。家庭などから通信事業者の光ファイバー回線などを通じてインターネット接続サービスを利用する際によく使用される。家庭内LANに接続されたパソコンや(ブロードバンド)ルータなどの機器と、インターネットサービスプロバイダ(ISP)の施設に設置された通信機器の間で接続を確立する手段としてよく利用される

- **VLAN**(*Virtual LAN*)\
1つの構内ネットワーク(LAN)内に物理的な接続形態とは独立に機器の仮想的なグループを設定し、1つのLANのように運用する技術。LANスイッチ(スイッチングハブ)の1機能で、接続された機器を管理者の設定した任意の数のグループに分け、ネットワークを分割できる。グループ範囲内でのみ直接通信ができ、ブロードキャストフレームによる回線の混雑を緩和できる

- **VPN**(*Virtual Private Network*)\
インターネット上に稼働的な専用網を設置し、安全にデータをやり取りする通信方式。距離が離れた拠点同士の場合、専用線を用意しての通信は多大なコストがかかる。VPN接続ならインターネットを利用し、論理的には専用線を繋いで通信しているのと同じ状態を実現できる\
データがインターネット上を伝送する前に暗号化され、復号の際はパスワードが必要。ユーザのデバイスからVPNサーバまで暗号化されたトンネルを通って送信され(カプセル化)、このセキュリティ対策によって外部ネットワークの影響を受けにくくなり、情報漏洩を防ぐ

- **FQDN(*Fully Qualified Domain Name*)**\
インターネットなどにおけるドメイン名の表記法の1つで、トップレベルドメインから順に、サブドメイン名やホスト名など各階層を省略せずに全て指定した形式のこと。IPアドレスではなく人間の覚えやすい英数字を組み合わせた識別名であるドメイン名を付与し、DNSでIPアドレスとの対応関係を管理している

- **ダウングレード**\
利用している機器やサービス、ソフトウェアなどを現在より格や等級の低いものに替えること。仕様変更により古い機器やデータ形式との互換性が失われるなど、利用に差し支える場合や、何らかの不具合が存在する場合がある

- **リストア**\
コンピュータに導入されたOSやアプリケーションソフトに不具合が生じ正常に起動・動作しなくなった場合、正常な状態に復元したり、初期状態に戻したりすること。ストレージ装置が破損するなどしてデータが失われた際に、以前に取っておいたデータの複製を書き戻すなどして復元すること。複製の場合はバックアップ

- **ホットフィックス(*hotfix*)**\
ソフトウェアに発見された不具合を解消するために配布される修正プログラムのうち、問題に迅速に対応するため緊急に発行されるもの。保安上の弱点を修正するためのものが多い

- **セッションタイムアウト**\
通信システムなどで、接続や利用の開始手続きをしてから交わされる一連の操作の連なりのこと。同一セッションの間は連続的にやり取りを行うことができ、操作の度に接続開始や認証の手続きを繰り返さずに済む。ログアウトなどの終了手続きによって明示的に終了・切断できるが、所定の手続きを行わずに利用者が去ったり通信が途絶することもあり、所定の時間だけ応答が無ければセッションは閉じられる

- **SOC**(*Security Operation Center*)\
情報システムへの脅威を、24365で監視し、脅威情報の分析などを行い、対応策を示す専門組織を指す。分析したアラート情報は、CSIRT(*Computer Security Incident Response Team*)や情報システム部門に連携され、本格対応が進められるパターンが一般的

- **CSIRT**(*Computer Security Incident Response Team*)\
コンピュータセキュリティに関わるインシデントに対応するチーム。インシデント発生後の被害拡大の防止や根本解決を主として対応を行うのに対し、SOCはインシデントが発生しないように事前に対応する点が異なる

- **EDR**(*Endpoint Detection and Response*)\
セキュリティ対策の一種で、端末の処理を常時監視して不審な挙動を検知し、記録を取って管理者に通報するもの。ネットワーク端末に存在するパソコンやサーバ、モバイル端末などに導入し、システム上に常に常駐してアプリケーションの振る舞いや内外のデータ送受信、システムへ加えられた変更などを監視する。セキュリティ上問題のある行為や通常の利用状況から大きく外れる不審な挙動を検知すると、マルウェア感染や攻撃者による遠隔操作などを疑い、動作ログなどの記録と共にシステム管理者に通知を行う\
従来はインターネット境界にFWやIDS/IPSを設置する境界防御が重視されてきたが、境界が流動化し標的型攻撃で乗っ取られた内部の端末から攻撃が行われるなど情勢の変化に対処しきれなくなっていた。内部の端末でも互いに無条件に信用しないゼロトラストの実現に貢献する

- **デプロイ**(*deploy*)\
ソフトウェアを構成するプログラムなどをパッケージなどにまとめ、使用環境に導入可能な状態に組み立てる工程(リリース)、実際の使用環境に展開して導入する工程(インストール)、プログラムを起動して所定の設定や手続きを行う操作や要求を受け付けられる状態にする工程(アクティベート)などを含む。配備済みのプログラムなどの修正・更新・入れ替え・更新などのための稼働停止(ディアクティベート)・不要なソフトウェアの撤去(アンインストール)など、運用開始後の工程を含む場合がある。開発や更新の済んだソフトウェア(の新しいバージョン)を運用環境のサーバに展開する工程や作業を指すことが多い

- **YAML**(*YAML Air't Markup Language*)\
構造的なデータ集合を簡素な文字列として表記できるデータ形式の1つ。ファイル拡張子は.yml、公式には.yamlが標準。XNLのような汎用のマークアップ言語に近いが、文法はプログラミング言語に近い

- **IRQ**(*Interrupt Request*)\
コンピュータ内部の装置や周辺機器などが中央処理装置(CPU)に信号を送り、現在の処理を中断して強制的に指定した処理を実行するよう要求すること(割り込み要求)

- **スティッキービット**(*sticky bit*)\
UNIX系OSのファイルシステムで設定される特殊なアクセス権の1つ。ファイルやディレクトリの所有者とスーパーユーザ(*root*ユーザ以外)、ファイルの改名や削除をできないようにする。多くのシステムでディレクトリにのみ設定できる

- ネットワーク構成\
**内部ネットワーク(*trust*)** : 企業の社内ネットワークなど(*internal*)\
**外部ネットワーク(*untrust*)** : インターネットなど(*external*)\
**DMZ(*DeMilitarized Zone*)** : どちらのネットワークからも隔離されたネットワーク。外部に公開するサーバを導入するネットワーク。サーバを不正に乗っ取られた場合にも内部ネットワークへの不正アクセスを防止できる

- **ZTNA**(*Zero Trust Network Access*)\
ゼロトラストに基づいたネットワークセキュリティの考え方のこと。社内外問わず全てのアクセスを制御することで、アカウント侵害や内部不正などのセキュリティリスクを低減する。社内外の境界線が曖昧になったことで、区別をせずに全て信頼しないというゼロトラストに基づいたセキュリティ技術が求められるようになった。VPNは接続時に1度IDやパスワードによって認証されれば、社内ネットワークの全てにアクセスできてしまう。攻撃者によるアカウント侵害があった場合、社内ネットワーク侵入後の不正アクセスの水平展開が起こりやすく、別の場所に保管されている重要な情報資産まで搾取されるリスクがある。実現に必要な機能の例は下記\
`ファイアウォール` : パケットやIPヘッダ値の正当性などを分析し、トラフィック制御を行う機能\
`Next Generation Firewall` : ポート番号やプロトコルに依存せず、アプリケーションを識別してトラフィック制御を行う機能\
`IPS/IDS` : 脆弱性を悪用した不正アクセスや攻撃を検知・防御し、端末やサーバを保護する機能

- **ディスククォータ(*disk quota*)**\
複数の利用者が共用するコンピュータで、1人ずつ割り当てられたストレージ(外部記憶装置)の上限容量。OSの管理機能として提供されるもので、管理者が各ユーザアカウントやディレクトリごとに自由に使える容量の上限値を設定できる

- **ロードバランサ**\
通信制御装置の1つで、外部から送られてくるデータや処理要求を、同等に機能する複数のシステムに振り分け、1台あたりの負荷を抑える装置。様々な種類があるが、ネットワーク上でサーバの負荷を分散するサーバロードバランサを指すことが多い。要求が増大してもシステムの数を増やすことで全体の対応能力を増強できる(スケーラビリティの向上)

- ***WildFire***(*Palo Alto*)\
統合クラウドマルウェア防御エンジン。機械学習モデルを利用して、未知の標的型マルウェアとAPT攻撃(標的型攻撃)をリアルタイムに検出する。サンドボックス環境で通過したファイルの自動分析を行い、悪意ある行動の有無を監視する

- **インスペクションモード**(*Fortigate*)\
`プロキシモード` : スループット(処理能力)よりセキュリティを優先させる。1度の送信・受信パケットを全てFGが組み立てて精査する。パケットを全て組み立てるためセキュリティレベルが高くなるが、1度バッファするためスループットが犠牲になり、バッファリング設計が複雑になる\
`フローモード` : セキュリティよりスループットを優先させる。パケットがグレーな状態で通過するため、パケットの一部から外部に情報が漏洩し、セキュリティレベルの低下に繋がる可能性がある

- ***Override***(*Fortigate*)\
オーバーライドが有効な場合、HA機能のAgeが長さより、プライオリティの大きさの評価が優先される。Ageは300s(デフォルト値)以内の誤差は同じ値として判断する

- **トラフィックログ**(*Fortigate*)\
IPv4ポリシーなどの設定したルールにより許可(拒否)した通信のログのこと。CLIでログのカテゴリの指定や表示ができる\
どのルータを通ってネットワークのどの部分でデータがどれだけやり取りされたかといった情報が含まれる。ネットワーク環境の監視やトラフィックの増減分析、ネットワーク障害の原因追求に使われることが多い

- **HA構成**(*Fortigate*の場合)\
*`Active - Active`* :ネットワーク負荷のバランスを取りやすく、トラフィックが急増してもスケールアウトによって対応できる。各デバイスがトラフィックの一部を処理するため、一部デバイスにトラフィックが集中せず安定した通信が行える。設定が複雑になりがちで、A - P構成よりも細かいチューニングが必要になる場合がある\
*`Active - Passive`* : 同じモデルのデバイスを複数用意し、1台を*Primary*(マスタ)、他を*Secondary*(スレーブ)として構成する。マスタが障害を検知した際は、自動的に系が切り替わり業務を継続する。システムの継続性が担保されるため、ネットワークのダウンタイムを減らすことが可能

- **UTM**(*Unified Threat Management* : 統合脅威管理)\
複数のセキュリティ機能、またはサービスをネットワーク内の単一のデバイスを統合することを指す。アンチウイルス、コンテンツフィルタリング、E メール・Webフィルタリング、アンチスパムなど、複数の異なる機能でネットワークのユーザを保護できる

- **LANSCOPE**\
企業のIT資産管理及びセキュリティ対策を支援するための統合管理ツール。組織内の端末状態の可視化や情報漏洩対策を主な目的としている。機器の稼働状況を網羅的に把握し、導入されているソフトウェアやライセンスの有効期限などの資産情報を自動で収集する。操作ログの収集、外部記憶媒体の利用制御、Webアクセスの監視などを通じて、不正利用や情報持ち出しの抑止を支援する。マルウェア対策、攻撃検知、監視代行サービスなどのエンドポイントセキュリティ機能や、クラウドサービスのログ監査、クラウドセキュリティの専門家による診断、ネットワーク脅威検知などの機能やサービスも提供している

- ***Vmware***\
1台のコンピュータ上で複数のOSやソフトウェアを動作させる仮想マシン・ソフトウェアを提供している企業。サーバ、クライアント(OS, アプリケーション)、クラウド環境を効率的に運用するための仮想化環境を構築する際にVMware製品が利用されている\
物理的な構成にとらわれずに論理的に統合・分割できる技術。複数のサーバは別々の物理サーバ上で稼働していたものを、1台の物理サーバ上で稼働させることが可能\
1台のサーバに集約するのでリソースを効率的に使用でき、運用負荷の軽減につながる。1つのファイルとして、ファイル感覚で管理することが可能(カプセル化)。管理画面上で通常のファイルと同様にコピーし、別サーバ上で稼働させたりバックアップ取得や複製が可能

- ***cloud***\
まとまった計算資源を通信ネットワークを介して遠隔から利用するシステム形態のこと。利用側から見て、ネットワークを通じて遠隔から計算資源を提供するシステムなどのこと。手元のコンピュータに導入して利用したソフトウェア、データ、提供のための機器や回線などの技術的基盤を、インターネットなどのネットワークを通じて利用者に提供する方式。大規模なDCなどに多数のサーバコンピュータや通信機器などを用意し、遠隔からインターネットなどの広域回線網(WAN)を経由して契約者にシステムを操作させる。利用者に登録後すぐに用意されたシステムの提供する機能や資源にアクセスして使用できる。購入はせず基本料金に使用実績や追加機能、容量に一定額を課金するフリーミアムモデルをとるものが多い。規模に合わせた増強や一時的な大量の資源の借り受けなどの柔軟な対応も可能な一方、コストの返答が大きい可能性がある

- ***on-premises***\
企業などの組織における情報システムの設置形態の分類で、自社施設の構内に機器を設置してシステムを導入・運用すること。元来この方式が一般的で名称が無かったが、通信ネットワークを通じて外部の事業者の設備を借用するクラウドコンピューティングが普及したため、区別のため名付けられた。設備を自社で用意するため、イニシャルコストが大きくなりがちで、稼働開始までの時間が長くなりがち。保守や管理、設備の更新も自前で行う

- **VDOM**(*Virtual Domains*)\
1台の機能を仮想的に複数台の独立した機器のように利用できるようにする*Fortigate*シリーズの機能の1つ。VDOMを有効にすると想的に複数台の独立したFG機器に分割でき、それぞれ別のネットワークに繋げて独立に運用できる。異なる用途のネットワーク毎に個別にUTMの導入が不要となり、1台に集約できる。各VDOM間はネットワーク的に仕切られて互いに通信できないが、設定により接続することは可能

- ***Session Initiation Protocol***(SIP)\
セッションの生成、変更、切断を行う飲みのプロトコルで、音声やビデオ通話などのリアルタイムコミュニケーションを可能にする

- ***Lightweight Directory Access Protocol***(LDAP)\
インターネットなどのIPネットワークを通じてディレクトリサービスにアクセスするためのプロトコルの1つ。ネットワーク上の利用者やサービス、周辺機器などの情報を一元的に管理できる

- **ディレクトリサービス**\
ネットワーク上のコンピュータの利用者やプリンタなどの周辺機器、共有フォルダなどのサーバ上のサービス、その他何らかの資源に関する情報をサーバ上のDBで一元管理する仕組み。利用者アカウントをディレクトリに登録し、複数のコンピュータで横断的に利用することができる

- **ONU(*Optical Network Unit*)** : 回線終端装置\
光ファイバーから送られてくる光信号をデジタル信号に変換する。光信号はパソコンで認識ができないため変換する必要がある。ONUは光信号を使ってインターネットに接続するが、モデムでは電話回線を通るアナログ信号を使ってインターネットに接続する。ルータはインターネットの回線を複数の端末が同時にインターネットを使えるようになる

- **RSTパケット(*reset*)**\
TCPで接続を中断・拒否する際に送られるパケット。クライアント(サーバ)がTCP通信の終了を一方的に通知する時に送信する。通常は何らかの異常を検知して通信を即座に終わらせたい時に使う

- **アノマリ検知**(*anomaly*)\
IDS, IPSが実施する不正アクセスを検知する方法。登録している範囲から外れた挙動を全て異常と検知する。閾値での判断ではなく過去のデータと比較して検知できる

- **SFPモジュール**(*Small Form - factor Pluggable* : 光トランシーバ)\
電気信号を光信号に変換する拡張モジュールで、スイッチングハブに接続して使用する。UTPケーブルでは100m程度の伝送距離を延長できる

- **ll**\
コマンドではなく、特定コマンドの別名として登録・定義された*alias*(別名)、一般的には`ls -l`の*alias*。ファイルの一覧や詳細を表示させる際に使用するコマンド

- **scp**(*Secure CoPy*)\
サーバ間でファイルの送受信を行うためのコマンド。通信プロトコルにはSSHが使われる。コピー元のローカルファイル、リモートサーバのユーザ名@ホスト名 : コピー先のディレクトリの順で指定する。-rだとディレクトリごと再帰的にコピー、-iだと秘密鍵ファイルのパスを実行する必要がある

- **デバイス証明書**(*Device Certificate*)\
ユーザが使う端末に登録される電子証明書。DCがインストールされたデバイスは正規にシステムを利用できる端末と認識され、クラウドサービスへのアクセスが許可される。第三者機関の認証局(*Certificate Authority*)が発行する

- **Syslogサーバ**\
ネットワーク上にある各ネットワーク機器(アクセスポイント、LANスイッチ、ルータ、ファイアウォールなど)の内部で起きている動作を収集し保存するサーバ。異なるメーカのネットワーク機器も一元的に解析できる標準のメッセージ形式を使用している

- **カナリアリリース**\
新バージョンのアプリケーションをリリースする際、一部のユーザに限定公開してテストを行い、問題なければ一般に向けて公開する手法。テストで不具合を確認できたり問題があっても戻せる一方、追加コストや労力がかかる特徴がある

- **リグレッションテスト**\
機能の追加・変更・不具合の改修などに伴うプログラムの変更によって、別のプログラムに意図しない不具合が発生していないか確認・検出するテスト

- **デバイスドライバ**\
コンピュータ内部に装着された装置や、外部に接続した機器などのハードウェアを制御・操作するためのソフトウェア。OSの一部として取り込まれて一体的に動作する。OSがハードウェアを制御するための橋渡しを行うプログラムで、利用者が直接操作することは稀で、OSに組み込まれて機能の一部として振る舞うようにできている。OSや各プログラムは定められた手順でデバイスドライバに処理を依頼する形を取ることで、それぞれが個別のハードウェアの制御仕様に直接対応する必要がなくなり、機種の違いに依らず同じ機能は同じ手順で利用できる\
個々のハードウェアは固有の機能や制御仕様を持っているため、原則機種毎に対応するデバイスドライバを入手・導入しなければ使用・操作できないが、キーボードやマウスなどの差異が小さい装置については共通仕様が定められている場合があり、OSに付属する標準のドライバで大半の機能を使用できることが多い

- **コントロールプレーン**\
制御情報とデータの伝送路が分かれているような機器やネットワークで、制御情報を送受信するための伝送経路。そのような経路を用いて実現される制御機能。要素間で制御情報を送受信し、設定や監視、指示の伝達などを行うための伝送路を指す

- **データプレーン**\
制御情報とデータの伝送路が分かれているような機器やネットワークで、データを送受信するための伝送経路。要素間で実質的なデータの受け渡しを行うための伝送路を指す

- **ARPクリア**\
ARPテーブル内には今まで学習した内容がキャッシュされている。定期的にリフレッシュされるため、しばらく通信がない場合はARPテーブル上のエントリ(データの項目・記録)から削除されるようになっている。リプレースを実施するとMACアドレスとIPアドレスの組み合わせが変わるため、整合性を保つためにARPテーブルの更新が必要となる

- **ネクストホップ**\
ルータなどが持つ経路表(ルーティングテーブル)に記載される情報の1つ。宛先ネットワーク毎に決められた、次に転送する隣接ルータ(IPアドレス)のこと

- **TTL**(*Time To Live*)\
機器間で送受信されるパケットや情報などの有効期限、破棄されるまでの残り時間などのこと。最大転送回数は最大255までの整数が設定され、ルータなどの機器を1度通過するごとに値が1減少することで、無限ループを防ぐ。0になったパケットはその場で破棄され、破棄通知が送信元に送られる。

- **ホールドダウンタイマー**\
経路情報がダウンした事を示すアップデートを受信した時、その経路情報を誤って再登録しないように、経路情報がダウンしたことを全てルータが認識できるようにするための待ち時間のこと。ルートポイズニングによる到達不能な経路情報(メトリック16)を受信した時、または一定時間(180s)を過ぎても隣接ルータから経路が通知されない時に作動し始める

- **ランレベル**\
*Linux*の動作モードのこと。0 ~ 6があり、どのようなサービスを起動する(しない)かを決定できる

- **SOA(*Service Oriented Architecture*)**\
企業の業務システムなどの設計様式の1つ。システム全体を利用者側から見たソフトウェアの機能単位であるサービスの組み合わせによって構築すること。コンピュータ寄りの視点での機能分割ではなく、利用者側の視点での作業単位に対応するように個々のサービス内容が決められる。ソフトウェアは独立性が高く依存性が低くなるよう設計され、柔軟に入れ替え・修正などに対応できる。汎用性や共通性の高いサービスは複数のシステムやアプリケーションから参照でき、同じような機能や部署がアプリケーション毎に重複して開発される事態を防ぐことができる

- **API**(*application programming interface*)\
ソフトウェアの機能や管理するデータなどを、外部の他のプログラムから呼び出して利用するための手順やデータ形式などを定めた規約のこと。多くのソフトウェアが共通して利用する機能は、OSやミドルウェアなどの形でまとめて提供されており、このような汎用的な機能を呼び出して利用するための手続きを定めたもの。開発者はAPIに従って機能を呼び出す短いコードを記述するだけで、処理内容を記述しなくても機能を利用したソフトウェアを作成できる。APIを経由して機能を呼び出す形でプログラムを構成することで、同じAPIが実装されていれば別のソフトウェア上でそのまま動作させることができる。

- **MAPI**(*Messaging API*)\
APIの1つで、電子メールなどのメッセージの送受信や管理などの機能にアクセスするためのもの。ソフトウェアからメッセージング機能を利用するための標準の呼び出し規約を定義しており、アドレス帳の利用、メッセージの送信、メールボックスへのアクセス、ディレクトリサービスへのアクセスなどが行える。LANなどのネットワーク上でMAPIに対応したクライアントとサーバが通信を行うためのプロトコルも定義されている

- **疎結合**\
複数のプロセッサが独立して動作し、相互に緩やかに通信を行うシステム。各プロセッサが自己完結型のタスクを処理し、障害が発生しても他のプロセッサに影響を及ぼしにくい。在庫管理や大規模データなど、複数のプロセッサが協力して作業を分担するシナリオで特に有用。プロセッサ間の通信を少なくできるため、スケーラビリティや耐障害性が向上する一方、ダイナミックな負荷分散には限界がある\
サービス同士をメッセージキューやAPIを介して通信させることで、各サービスを独立して開発・運用できる。汎用的な端子を通じて異なるメーカーや世代の機器を組み合わせられる状態は、物理的な結びつきが緩やかであることを示す

- **密結合**\
複数のプロセッサが高い速度で相互にデータをやり取りできるように設計されたシステム。各プロセッサは共有メモリを通じて通信し、協力してタスクを処理する。スーパーコンピュータがこのシステムを利用することで、大規模な計算を迅速に行うことが可能。プロセッサ間の通信が迅速であるため、並行処理の効率が高まる一方、システムのスケーラビリティやコストの面で挑戦が伴うこともあるが、高負荷の計算を扱う環境において効果的な技術

- **モノリシック(*monolithic*)**\
要素が分割されておらず全体が一体になっている様子。構成要素の独立性が低く、一部分だけを分離・交換できるように作られておらず、全体が緊密に一体化している。特定の部分を切り離したり、外部から他の要素を追加・交換したりすることは想定していないような設計。単体としての処理効率などは向上させやすいが、柔軟性や拡張性は犠牲となる

- ***daemon***\
UNIX系OSでメインメモリ上に常駐して特定の機能を提供するプログラム。利用者の操作とは無関係に処理を行うバックグラウンドプロセスの一種で、別のプログラムや機器、通信相手など外部からの要求を受け付けて処理を実行したり、予め設定されたタイミングで自律的に特定の処理を実行したりする。実行可能ファイルの名称には、処理内容を表す単語の末尾にdをつける習慣がある

- **rwx**\
*read*(読み取り), *write*(書き込み), *execute*(実行)のアクセス権が設定され、ユーザごとにファイルへのアクセスの許可 / 禁止を設定できる。アクセス権は2進数で表され、全権限は7となる

- ***proxmox***\
仮想マシンとコンテナをGUIから一元管理できる、OSSの仮想化プラットフォーム。ライブマイグレーション、高可用性、仮想マシンFWなどの機能を備えている

- **カーネル**\
OSの中核となるソフトウェア。動作中のプログラムの実行状態を管理したり、ハードウェア資源を管理してプログラムがハードウェアの機能を利用する手段を提供したりする。コンピュータ起動時にブートローダなどによって一番最初に起動され、特権モードなど強い権限を持つ特別な動作モードで実行される。電源弾や再起動まで常駐し続け、プログラムの起動や終了後の後始末を行う

- **tar(*tape archives*)ファイル**\
複数のファイルを1つにまとめて格納するアーカイブファイルの形式の1つ。利用者の指定した複数のファイルを連結した1つのファイルを作成することができ、多数のファイルで構成されるデータやプログラムなどを装置間や利用者間、コンピュータ間で複製、配布などの際、1つずつ指定しなくてもまとめて扱える

- **tgzファイル**\
圧縮ファイルの形式の1つで、tarコマンドで複数のファイルやディレクトリをアーカイブ化し、gzip形式で圧縮したもの。tarコマンドは複数ファイルやディレクトリを属性やディレクトリ構造を保ったまま1ファイルにまとめることができ、一体化したファイルをアーカイブファイル(*archive file*)と呼ぶ\
tarコマンドで作成したアーカイブファイル(tarファイル)をgzipコマンドでデータ圧縮し、容量を削減したものが.tgzファイルで、伸長すると元になったtarファイルを取り出すことができる\
tarとgzipは本来異なるコマンドだが、-zオプションでアーカイブ化と圧縮を同時に行うことができる

- **whereisコマンド**\
コマンド本体及びマニュアルファイルとソースファイルのパスを表示する。似た機能の*which*コマンドは本体のパスを表示し、*type*はコマンド名でbashが何を実行するかを表示する

- **仮想IPアドレス(*Virtual IP address*)**\
複数の機器で共有されるIPアドレスのこと。システムの冗長化や負荷分散などのために用いられる。1つの機器内の複数の装置で共有する場合もある。複数の機器をグループ化して1つのIPアドレスに紐付け、他の機器に対して1つの機器のように振る舞うことができる。2台のゲートウェイに同じVIPを割り当てると、切り替えを行わなくても常にどちらかのゲートウェイが接続を受け付ける状態を維持でき、稼働中の機器に接続できるので接続先を切り替えが不要

- **仮想MACアドレス**\
通常は装置毎に物理的に固定されているMACアドレスを、ソフトウェアによる制御により動的に割り当てたもの。ルータなどの機器が特殊な動作モードで使用するもので、使用時に一定の生成規則に基づいて割り当てられる。物理的なMACアドレスは重複できないが、複数機器で同じMACアドレスを共有して自動的に切り替えたい場合などに用いる。デフォルトゲートウェイを冗長化するプロトコルを使用する場合、2台のルータが同じ仮想MACアドレスをもち、普段はマスター機のみがその仮想MACアドレスへの送信に応じる。マスターが停止すると自動的にバックアップ機が有効になり、同じ仮想MACアドレスで通信するため、切り替わりを意識することなく通信を継続できる

- ***Source NAT***\
2つのIPネットワークの境界にある機器が双方のIPアドレスを自動的に変換するNAT(*Network Address Translation*)のうち、送信元アドレスを書き換える方式

- **ハートビート**\
外部から機器が正常に動作しているか常に調べ続ける死活監視の手法の1つで、監視対象の機器に予め決まった周期で短いデータを送るよう設定し、監視側では途切れず着信しているかを確認する。一定時間着信が途絶えると、相手側に何らかの不具合が生じたと判断し、管理者への通知や待機系への切り替えなど、事前に準備された対応を実行する。監視する側とされる側が固定されている場合と、対等な関係の機器同士が相互に監視し合う場合がある

- **RFC(*Request for Comments*)**\
国際基準で定められている。違反しているメールアドレスには、携帯電話同士では送受信ができても、プロバイダやレンタルサーバからは送信できないため、ワンタイムパスワードメールなどが届かない可能性がある。アドレスの先頭のドット(.)、@直前のドット、ドットの連続、半角英数字と一部の記号以外の文字列を含むと違反ケースとなる

- **RDP(*Remote Desktop Protocol*)**\
サーバコンピュータの画面をネットワークを通じて別のコンピュータに転送し、利用者が遠隔のコンピュータを手元にあるかのように操作できる技術。利用者が操作するローカル側とネットワークの向こうにあるリモート側との間の通信に用いるデータ形式や伝送手順を定めている。リモート側の画面情報をリアルタイムでローカル側に送信したり、ローカル側でキーボードやマウスなどを用いて操作・入力した情報をリモート側に伝達できる

- **RaaS(*Ransomware as a Service*)**\
ランサムウェアをサービスとして提供するもの。感染したPCのロックや、ファイルの暗号化により業務にインパクトを与え、暗号解除と引き換えに身代金を要求する。身代金を払う被害者が一定数いることでビジネスとして成立してしまい、また高度な技術を持たない人でも対価を払うことで簡単に攻撃を行える環境が整っていることで市場が成立している。開発者自らの手で攻撃を行うのが主流だったが、*RaaS*では開発者ではなく実行犯(アフィリアエイター)を集め、ランサムウェアを提供し、身代金が支払われると成功報酬として山分けされる

- **Tor(*The Onion Router*)通信**\
インターネット上で発信元の秘匿性が高い通信を行うことができるシステム。インターネット上のデータは小さな単位(パケット)に分割され、アドレスを付与して送り出される。このパケットは沢山のルータをバケツリレーのようにして届けられる。本体のデータは暗号化できても、送信先アドレスは無数のルータが読んで中継するため、暗号化ができない。\
Torでは世界中の協力者のサーバが中継して、目的地までパケットを届ける。利用時はノードが3つランダムに選ばれ転送されていく。転送経路を逆に辿りながらアドレスを含めて多重に暗号化していく

- **ダークパターン**\
ユーザを無意識のうちに特定の行動に誘導、または心理的な操作などを通じて不利益な選択をさせる可能性があるデザインやサービス設計のこと。追加料金の表示が購入手続きの最終画面で表示される、複雑な退会手続き、即座に行動を取らせるための焦るを感じさせる手法などがある

- **CPU**(*Central Processing Unit*)\
コンピュータの主要な構成要素で、他の装置・回路の制御やデータ演算を行う。論理演算や算術演算を行う演算ユニット(`演算`)、データの一時的な記憶を行うレジスタ(`保存`)、外部との通信を行うインターフェイス回路(`判断`)などで構成される。様々な種類の複雑なタスクを順序よく連続処理することが得意、複雑な演算とプログラムの制御を担当し司令塔のような役割を果たす。汎用性の高さと処理性能の高さを備えている一方、同時に複数の処理をすることは得意ではない

- **GPU**(*Graphics Processing Unit*)\
画面表示や画像処理に特化したIC(集積回路)。3DCG描画や動画の圧縮・展開などに必要な演算を高速化する並列処理に優れた構造のもの。単純な数値計算に特化した演算回路を大量に内蔵しており、1度に数百から数千の演算を一気に実行できる。3次元から2次元への座標値の変換(ジオメトリ処理)、立体表面に貼り付けられた画像パターン(テクスチャ)や光、陰影などから各画素の表示色を決定する処理(シェーディング)などを高速に実行できる。命令を1つずつ順に処理するCPUに対し、同種の計算を一斉に実行する並列処理がGPUの特徴(判断はしない)

- **TPU**(*Tensor Processing Unit*)\
大量の`行列演算`を高速に実行できる。乗算と加算を組み合わせた積和演算を実行する演算ユニットを大量に内蔵しており、多数の数値を並列に研鑽できる。HBM(*High Bandwidth Memory* : メモリ)を内蔵しており、メインメモリから読み込んだ行列データを高速に演算装置に割り当てることができる。AI処理速度が高速で、電力効率も高い

- **ホットスワップ**\
機器の電源が入り稼働状態のまま、部品やケーブルなどを装着・交換・抜去すること、またそのような仕組みやコネクタなどの構造のこと。機器を稼働させたまま部品の取り付けや取り外しを行い、取り付けた部品は自動的に認識され、すぐ制御可能になる。本体やコネクタなどのハードウェアが対応しており、かつファームウェアやOS、制御用のソフトウェアもホットスワップを前提とした設計や仕様の必要がある

- **ストレージ**\
データを永続的に記憶する装置。磁気ディスクや光学ディスク、フラッシュメモリ装置、磁気テープなど。通電しなくても記憶内容が維持される記憶装置を指し、コンピュータが利用するプログラムやデータなどを長期間に渡って固定的に保存したり、他の機器へのデータ運搬や複製、配布などのために用いられる\
メモリに比べて記憶容量は数十-数千倍大きく、容量当たりのコストが小さいが、読み書きに要する時間が大きい\
様々な動作原理の装置があり、磁気を利用するもの、レーザ光を用いるもの、電荷を利用するものに分けられる

- **レジスタ**\
CPUなどのプロセッサ内部にある、演算や実行状態の保持に用いる記憶素子。最も高速な記憶装置だが、一般的なCPU製品で数個から数十個、数十バイト程度。GPUなど特殊なプロセッサでは数千から数万(数百キロバイト)を内蔵するものもある\
CPU内部の演算回路などは原則としてレジスタに記録されたデータに対して処理を行うようにできている。コンピュータプログラムはメインメモリ(RAM)からレジスタにデータを読み出し、処理を行い、結果をメインメモリに書き出すという手順を繰り返して処理を進める\
処理対象のデータや演算結果、途中経過などが記録され、読み込みや書き出しを行いメインメモリ上のアドレス(番地)を指し示すデータを記録するのにも使われる。メインメモリ内の記憶素子のように番地によって識別されるのではなく、個別の識別名が与えられている\
命令によって役割が決まっている`専用レジスタ`、プログラムの都合で様々な用途に使いまわせる`汎用レジスタ`、プログラムからアクセスできず、プロセッサ自身が内部的に使用するための特殊なレジスタが用意されている`内部レジスタ`がある
